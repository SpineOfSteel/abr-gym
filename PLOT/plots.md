# Plots and Evaluation

This page documents the evaluation plots generated by `plot_grouped.py`
for the Norway mobile bandwidth dataset. The script groups logs by
transport mode, parses per-chunk measurements from the ABR client logs,
and computes four core metrics for each adaptation scheme: mean bitrate,
bitrate smoothness, stall percentage, and mean QoE/reward. It then
renders comparison line plots for each metric and a QoE CDF for the same
set of runs. The evaluated schemes are `bb`, `bola`, `mpc`, `rl`, `ppo`,
and `netllm`, shown in the plots as BBA, BOLA, RobustMPC, Pensieve,
Pen-PPO, and NetLLM, respectively.

The plotting code uses the following transport groups: `tram`, `car`,
`bus`, `ferry`, `metro`, and `train`. For the combined view, the script
subsamples traces more aggressively to keep the figures readable, while
per-transport plots preserve nearly all runs. In the code, bitrate is
reported in Mbps, smoothness is computed as the mean absolute change
between adjacent bitrate decisions, stall is summarized as a percentage
of session time spent rebuffering, and QoE is represented through the
reward column already logged by each scheme. These implementation
details come directly from `plot_grouped.py`.

## Dataset Background

The underlying bandwidth traces come from the MMSys 2013 Norway mobile
streaming dataset. The dataset groups logs by route and transportation
mode, and includes route maps plus observed bandwidth profiles. The
original measurements were collected during TCP streaming sessions over
Telenor's 3G/HSDPA network in Norway, with 2-second video segments. The
dataset page also notes that some routes contain tunnel segments with
much weaker signal conditions, which is particularly relevant when
interpreting sudden throughput collapse, elevated stalls, or
conservative bitrate choices in the transport-specific plots.

A useful way to read the evaluation is therefore to connect three views
for each transport: the route map, the measured bandwidth profile, and
the ABR outcome plots. The map explains the physical path, the bandwidth
graph summarizes the connectivity envelope along that path, and the ABR
comparison figures show how each controller reacts to those conditions
in terms of delivered quality, stability, interruptions, and overall
QoE.

## How to Read the Plots

### Average bitrate per trace

The bitrate plot compares the mean delivered video bitrate over each
trace. Higher values generally indicate that a scheme is exploiting
available bandwidth more aggressively. However, bitrate alone is not
sufficient: an algorithm can achieve high bitrate by being overly
optimistic, which may later show up as poor smoothness, more stalls, or
a weaker QoE distribution.

### Bitrate smoothness per trace

The smoothness plot measures the mean absolute difference between
adjacent bitrate selections. Lower values are better because they
indicate more stable visual quality and fewer disruptive up/down
switches. A scheme with only slightly lower bitrate but much better
smoothness may still provide a better user experience than a more
aggressive but oscillatory alternative.

### Time spent on stall per trace

The stall plot summarizes rebuffering burden for each trace. In the
script, this is computed from the cumulative rebuffer measurements in
the logs and normalized by session duration. Lower values are better,
and spikes often indicate severe bandwidth dips, poor startup
estimation, or a control strategy that overshoots the network capacity.

### QoE CDF

The QoE plot is shown as a cumulative distribution function over
per-trace reward. This is one of the most informative summary figures
because it captures both central tendency and tail behavior. Curves
shifted to the right indicate better reward values overall. A scheme
with a strong right-shifted CDF is typically balancing bitrate,
smoothness, and stall penalties more effectively across heterogeneous
traces.

## Transport-specific Evaluation

Each transport section should ideally contain six images:

1.  route map,
2.  route bandwidth profile,
3.  bitrate comparison,
4.  smoothness comparison,
5.  stall comparison,
6.  QoE CDF.

The route artifacts are especially important because the Norway dataset
is geographically grounded. Different modes of travel expose the client
to different connectivity regimes: tunnels, dense urban handoffs,
shoreline paths, rail corridors, open roads, and commuter patterns.
Those route-specific properties explain why the ABR plots vary so much
across transport categories.

## Implementation Notes from `plot_grouped.py`

A few implementation details are worth surfacing in the documentation
page because they influence how readers interpret the figures:

-   The script searches `*.txt` logs under the trace root and filters
    transport-specific files by checking whether `_{transport}_` appears
    in the filename stem.
-   Only files whose names contain one of the configured schemes are
    aggregated into the metric tables.
-   The bitrate plot reports the mean of the logged bitrate column after
    converting from kbps to Mbps.
-   The stall metric is derived from the rebuffer column and normalized
    as a percentage of playback-plus-rebuffer time.
-   The smoothness metric is computed as the mean absolute difference of
    adjacent bitrate samples.
-   The QoE CDF uses the logged reward values directly and plots their
    empirical cumulative histogram.

These details make the page easier to trust and reuse, especially for
readers trying to reproduce the figures or adapt the plotting code to a
new ABR benchmark.
